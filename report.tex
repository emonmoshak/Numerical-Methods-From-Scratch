\begin{abstract}
This project, \textit{Numerical Methods From Scratch}, presents a rigorous implementation of fundamental numerical algorithms using the modern C++20 standard. The primary objective was to build a scientific computing library from first principles, focusing on mathematical derivation, numerical stability, and computational efficiency. The report details the implementation of root-finding techniques, linear system solvers, interpolation methods, numerical calculus, and ordinary differential equation (ODE) integrators. By leveraging C++20 features such as template metaprogramming, RAII, and the Eigen library for linear algebra primitives, the resulting toolkit provides a robust and transparent platform for engineering analysis. Every algorithm is subjected to rigorous error analysis and validated against theoretical convergence rates, demonstrating the efficacy of the first-principles approach in scientific software development.
\end{abstract}

\section{Introduction}
Numerical analysis is the study of algorithms that use numerical approximation for the problems of mathematical analysis. In modern engineering, these methods are the backbone of simulation software, control systems, and data analysis tools. While high-level environments provide pre-built solvers, a deep understanding of the underlying mechanics is essential for ensuring stability and accuracy in sensitive applications such as robotics or aerospace engineering.

This report documents the design and development of the \textit{Numerical Methods From Scratch} library. The project bridges the gap between abstract mathematical theory and high-performance implementation. By using C++20, we achieve high efficiency while maintaining clear, type-safe abstractions. The library is designed to be modular, transparent, and educationally valuable, exposing the convergence history and internal state of each solver.

\section{Root Finding Algorithms}
Root-finding involves determining $x$ such that $f(x) = 0$. These methods are iterative and start with an initial guess or interval.

\subsection{Bisection Method}
The Bisection method is based on the Intermediate Value Theorem. If a continuous function $f(x)$ changes sign over an interval $[a, b]$, then a root must exist within that interval. The algorithm repeatedly halves the interval:
\begin{equation}
    c = \frac{a+b}{2}
\end{equation}
If $f(a) \cdot f(c) < 0$, the root is in $[a, c]$; otherwise, it is in $[c, b]$. The method is robust and guaranteed to converge, albeit linearly with a convergence rate of $0.5$.

\subsection{Newton-Raphson Method}
Newton's method uses a first-order Taylor series expansion to approximate the function linearly at the current point $x_n$:
\begin{equation}
    x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}
\end{equation}
This method exhibits quadratic convergence ($e_{n+1} \approx C e_n^2$), making it extremely fast when the initial guess is close to the root. However, it requires the existence and calculation of the analytical derivative.

\subsection{Secant Method}
The Secant method approximates the derivative using two previous points, eliminating the need for an analytical $f'(x)$:
\begin{equation}
    x_{n+1} = x_n - f(x_n) \frac{x_n - x_{n-1}}{f(x_n) - f(x_{n-1})}
\end{equation}
It has a superlinear convergence rate ($\phi \approx 1.618$).

\section{Linear Algebra Solvers}
Solving large systems of linear equations $Ax = b$ is a fundamental requirement in finite element analysis and optimization.

\subsection{Direct Methods: LU Decomposition}
Direct methods find the exact solution (within machine precision) in a finite number of steps. LU Decomposition factors $A = LU$ using Doolittle's algorithm. Once factored, the system is solved in two steps:
\begin{enumerate}
    \item Forward substitution: $Ly = b$
    \item Backward substitution: $Ux = y$
\end{enumerate}
This is particularly efficient for solving systems with multiple right-hand side vectors.

\subsection{Iterative Methods: Jacobi and Gauss-Seidel}
Iterative methods are preferred for large, sparse systems. They start with an initial guess $x^{(0)}$ and refine it. The Gauss-Seidel update rule is:
\begin{equation}
    x_i^{(k+1)} = \frac{1}{a_{ii}} \left( b_i - \sum_{j < i} a_{ij} x_j^{(k+1)} - \sum_{j > i} a_{ij} x_j^{(k)} \right)
\end{equation}
Convergence is guaranteed if the matrix $A$ is strictly diagonally dominant.

\section{Interpolation and Approximation}
Interpolation seeks a function $P(x)$ that passes exactly through a set of points $(x_i, y_i)$.

\subsection{Polynomial Interpolation}
We implemented Lagrange and Newton forms. While useful for small datasets, high-degree polynomials can suffer from Runge's phenomenon, where oscillations occur at the boundaries.

\subsection{Cubic Spline Interpolation}
Cubic splines solve the oscillation problem by using piecewise cubic polynomials $S_i(x)$ that are $C^2$ continuous (continuous first and second derivatives at the knots).
\begin{equation}
    S_i(x) = a_i + b_i(x-x_i) + c_i(x-x_i)^2 + d_i(x-x_i)^3
\end{equation}
This results in a smooth curve that is highly stable for large datasets.

\section{Numerical Calculus}
Approximating derivatives and integrals is necessary when analytical forms are unavailable or discrete data is provided.

\subsection{Numerical Differentiation}
We utilize finite difference quotients. The Central Difference scheme provides second-order accuracy:
\begin{equation}
    f'(x) \approx \frac{f(x+h) - f(x-h)}{2h} + \mathcal{O}(h^2)
\end{equation}

\subsection{Numerical Integration (Quadrature)}
Integration is performed using Newton-Cotes formulas (Trapezoidal, Simpson's) and Gaussian Quadrature. Simpson's 1/3 rule provides fourth-order accuracy for smooth functions:
\begin{equation}
    \int_a^b f(x) dx \approx \frac{h}{3} \left[ f(a) + 4 \sum_{i \text{ odd}} f(x_i) + 2 \sum_{i \text{ even}} f(x_i) + f(b) \right]
\end{equation}
Gaussian Quadrature chooses optimal points to integrate polynomials of degree $2n-1$ exactly with only $n$ points.

\section{Ordinary Differential Equations}
Initial Value Problems (IVPs) $y' = f(t, y)$ are solved by stepping through time.

\subsection{Euler and Improved Euler}
The standard Euler method is first-order, while Improved Euler (Heun's) uses a predictor-corrector approach to achieve second-order accuracy.

\subsection{Runge-Kutta 4th Order (RK4)}
RK4 is the most widely used integrator due to its excellent balance of computational cost and accuracy.
\begin{align}
    k_1 &= f(t_n, y_n) \\
    k_2 &= f(t_n + h/2, y_n + h k_1/2) \\
    k_3 &= f(t_n + h/2, y_n + h k_2/2) \\
    k_4 &= f(t_n + h, y_n + h k_3) \\
    y_{n+1} &= y_n + \frac{h}{6}(k_1 + 2k_2 + 2k_3 + k_4)
\end{align}
The error per step is $\mathcal{O}(h^5)$, and the global error is $\mathcal{O}(h^4)$.

\section{Engineering Architecture}
The software is designed using modern C++ principles. A core feature is the \texttt{nm::common::Result<T>} structure, which ensures that every computation is accompanied by its metadata.

\begin{lstlisting}[caption=Definition of the Result structure]
template <typename T>
struct Result {
    T value;                    // Computed numerical result
    Status status;              // Success, Diverged, MaxIterationsReached
    int iterations;             // Total iteration count
    std::vector<double> errors; // Error history for convergence analysis
};
\end{lstlisting}

By utilizing template specialization and the Eigen library, we ensure that the solvers are generic and can handle both scalar (double) and vector (Eigen::VectorXd) inputs seamlessly.

\section{Discussion and Lessons Learned}
The development of this library revealed several critical insights into scientific computing:
\begin{itemize}
    \item \textbf{Stability vs. Speed:} The Bisection method, despite being the slowest, was the most reliable solver. In engineering, ensuring a solution exists is often more critical than finding it quickly.
    \item \textbf{Error Propagation:} In ODE solvers, small local errors aggregate into significant global errors. High-order methods like RK4 are not just "better," they are necessary for long-term simulations.
    \item \textbf{Matrix Conditioning:} For linear systems, iterative solvers are sensitive to the condition number of the matrix. Diagonal dominance is a powerful predictor of success.
\end{itemize}
